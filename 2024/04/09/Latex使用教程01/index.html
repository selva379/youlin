Latex使用教程 第01章

介绍

基础概念

Latex是一套排版系统，包包括两部分，一个是编译的引擎，一个是编辑器。以贪吃蛇游戏作为类比，前者就像游戏软件本身，后者则是外部的按键摇杆，通过操作调整后者来对前者进行控制改变。与实际游戏中所见即所得的模式不同，latex中的操作所得到的效果并不会立即显现，而需要编译之后才能呈现，如同蒙着眼睛操作贪吃蛇，编译相当于最后去掉眼罩去观看屏幕上的最终成果。

如果和常用的文字排版工具word做类比，后者是所见即所得的效果，以及操作是通过图形界面，而Latex的操作则是通过代码界面，这二者决定了学习使用Latex的不便与高门槛，那为何还要使用，可能是对于公式比较好的支持吧。（此处还没完全想好）

回到Latex的两部分，第一部分在网络上可以自由下载，常用的发行版有 TeX Live、MikTeX 和适用于 macOS 用户的 MacTeX（实际上是 TeX Live 的 macOS 版本）。TeX Live 和 MacTeX 带有几乎所有的 LaTeX 宏包；而 MikTeX 只带有少量必须的宏包，其他宏包将在需要时安装。何谓宏包？在计算机编程和软件开发的语境中，“宏包”（Macro package）通常指一系列宏的集合。宏是一种指令，它会被自动扩展或转换成一组指令，用以自动化重复性的任务或者将复杂的代码简化。宏包通常用于特定的程序或软件中，提供一组预定义的宏，以帮助用户更高效地完成特定的编程或文档编辑任务。在TeX或LaTeX这类排版软件中，宏包尤其常见。在LaTeX中，宏包提供了额外的命令和环境，用于处理诸如复杂表格、数学公式、参考文献引用等特定格式或功能。这些宏包极大地扩展了LaTeX的功能，使用户能够更容易地实现复杂的排版需求。例如，“graphicx”宏包提供了插入和处理图像的功能，“biblatex”宏包提供了先进的参考文献管理工具。因此，宏包在提高工作效率、实现特定功能和自动化任务方面发挥着重要作用，尤其是在需要精确和专业排版的场合。

没有宏包，latex功能的丰富性和便捷性会大打折扣，因此在之后的学习中，知道有哪些比较基本和常用的宏包以及能够熟练使用它们是重点内容之一。

安装方法

对于 Windows 用户，你需要下载 TeX Live 或 MikTeX。国内用户可以使用清华大学 TUNA 镜像站，请点击页面右侧的「获取下载链接」按钮，并选择「应用软件」标签下的「TeX 排版系统」即可下载 TeX Live 或 MikTeX 的安装包，其中 TeX Live 的安装包是一个 ISO 文件，需要挂载后以管理员权限执行 install-tl-advanced.bat。

对于 macOS 用户，清华大学 TUNA 镜像站同样提供 MacTeX 和 macOS 版 MikTeX 的下载。

对于 Linux 用户，如果使用 TeX Live，则同样下载 ISO 文件，执行 install-tl 脚本；如果使用 MikTeX，则按照 官方文档 进行安装。

文档结构

基本要素

在纸面上，一篇比较基本的文档一般由三部分构成：标题、作者和正文。而要通过Latex实现相应的排版效果，也需要不同部分或者要素的组成，只不过其要素更多，以及混杂了很多代码指令在其中，不过不用担心，通过由简到繁的逐步梳理，最终读懂和学会排列这些要素都不是难事。

Latex文档的要素放在一个后缀为tex的文件当中，创建tex文件一共有两种方法，第一是直接在编辑器当中新建，第二则是新建txt文本文档，并将txt后缀改成tex后缀，再双击打开进行编辑。

现在我们获得了一个空白的tex文档，在进行正式的编辑操作之前，需要进行一些设置以方便之后的操作，首先在编辑器选项菜单中，勾选显示行号，不同的编辑器选项菜单的位置不同，但总体的思路来说还是选项-设置-编辑器。行号并不是要素，但它可以帮助你比较代码与屏幕信息，找到错误，就像狙击枪上的瞄准镜一样。如果默认没有开启高亮，就进入设置菜单开启代码高亮，会使得代码更加易读，就像参考线一样。

页眉页脚

页码

这一部分最常见的需求就是在页脚添加页码。默认情况下，LaTeX会在每个页面的底部生成一个页码编号，即使是标题页也是如此。通常情况下，我们不需要在标题页输出页码编号，而且目录页一般也会采用罗马数字的形式给出页码，而正文内容则以阿拉伯数字的形式给出页码编号。

在LaTex中，与页码和页面设置相关的命令有pagestyle{type}，thispagestyle{type}，setcounter{page}{number}，pagenumbering{style}。
pagestyle{type}和thispagestyle{type}命令用于设置页面样式（页眉和页脚）。pagestyle{type}和thispagestyle{type}的主要区别在于pagestyle{type}对后续所有页面都有效，而thispagestyle{type}仅仅对当前页面有效。在LaTeX中，默认提供了以下四种页面样式：

empty - 没有页眉和页脚
plain - 没有页眉，页脚包含一个居中的页码
headings - 没有页脚，页眉包含章/节或者字节的名字和页码
myheadings - 没有页脚，页眉包含有页码

问题来了，Preface部分通常并算作章节的内容，也就不会作为起始的章节1，那么如何才能实现这一效果，并且实现Predace标题的居中呢？目前我也不知道，留待日后解决吧。

查到了：当你使用带星的分节命令(例如，chapter*{Preface})时，这些章节就不会出现在目录里。[1] 这种情况下还需要解决的问题就是，如何单独调整这一章标题的字体字号、对齐等信息呢？

默认情况下，页码从第一页开始（通常是包含标题页的）并且以阿拉伯数字形式显示（例如：1，2，3 …）。页码的编号也可以通过setcounter{page}{number}命令手动设置，需要注意的是，手动设置可能会导致重复的页码出现。
pagenumbering{style}命令则用于改变页码的显示风格。LaTeX中页码风格包含以下五类：

arabic - 阿拉伯数字
roman - 小写的罗马数字
Roman - 大写的罗马数字
alph - 小写的字符形式
Alph -大写的字符形式

生成目录（TOC）

如果你使用分节命令，那么可以容易地生成一个目录。使用 \tableofcontents 在文档中创建目录。通常我们会在标题的后面建立目录。

你可能也想也想更改页码为罗马数字（i,ii,iii）。这会确保文档的正文从第 1 页开始。页码可以使用 \pagenumbering{...} 在阿拉伯数字和罗马数字间切换。

注释

\footnote命令是LaTeX中创建脚注的主要命令，它主要由两种使用形式：

  • \footnote{text for footnote}：这一命令会自动在文本中插入一个上标数字，被称为脚注的标号（mark），同时在页面的底部插入脚注的标记和内容。
  • \footnote[number]{text for footnote}：这一命令接受一个额外的参数number来指定脚注的标号。
